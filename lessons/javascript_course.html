1.2 Console & Variables

Learn the basics of outputting text and storing data in variables - the building blocks of JavaScript programming.

Console Output

The console.log() function is used to output text to the browser's console.

console.log("Hello, World!");
Hello, World!

Welcome to JavaScript!

Variables

Variables are containers for storing data values. JavaScript has three ways to declare variables.

// Using let (recommended for variables that change)
let name = "Alice";
let age = 25;
let height = 1.75;

// Using const (for values that don't change)
const PI = 3.14159;

// Using var (older way, avoid in modern code)

// String

let isFalse = false;

1.3 If-Else

Learn how to make decisions in your code using conditional statements. This is one of the most fundamental concepts in programming.

What You'll Learn

  • Basic if statements and conditions
  • If-else and if-else if-else structures
  • Comparison operators and logical operators
  • Ternary operators
  • Switch statements

Understanding Conditions

Conditions are expressions that evaluate to either true or false. JavaScript uses these boolean values to decide which code to execute.

// Simple conditions
let x = 5;
let y = 10;

console.log(x > y);
false

console.log(x < y);
true

console.log(x === 5);
true

Comparison Operators

JavaScript provides several operators to compare values:

// Comparison operators
let a = 10;
let b = 5;

console.log(`a > b: ${a > b}`);
a > b: true

console.log(`a < b: ${a < b}`);
a < b: false

console.log(`a === b: ${a === b}`);
a === b: false

console.log(`a !== b: ${a !== b}`);
a !== b: true

console.log(`a >= b: ${a >= b}`);
a >= b: true

console.log(`a <= b: ${a <= b}`);
a <= b: false

Basic If Statement

The if statement allows you to execute code only when a condition is true.

let age = 18;

if (age >= 18) {
  console.log("You are an adult");
  console.log("You can vote!");
}

You are an adult
You can vote!

If-Else Statement

Use else to provide an alternative when the condition is false.

let temperature = 25;

if (temperature > 30) {
  console.log("It's hot!");
  console.log("Stay hydrated!");
} else {
  console.log("It's comfortable");
  console.log("Enjoy the weather!");
}

It's comfortable
Enjoy the weather!

If-Else If-Else Statement

Use else if to check multiple conditions in sequence.

let score = 85;

if (score >= 90) {
  grade = "A";
} else if (score >= 80) {
  grade = "B";
} else if (score >= 70) {
  grade = "C";
} else if (score >= 60) {
  grade = "D";
} else {
  grade = "F";
}

console.log(`Your grade is: ${grade}`);

Your grade is: B

Logical Operators

Combine multiple conditions using logical operators: && (AND), || (OR), and ! (NOT).

let age = 25;
let hasLicense = true;
let hasCar = true;

// Using AND operator
if (age >= 18 && hasLicense) {
  console.log("You can drive!");
}

// Using OR operator
if (hasCar || hasLicense) {
  console.log("You have transportation options");
}

// Using NOT operator
if (!hasLicense) {
  console.log("You need to get a license first");
}

You can drive!
You have transportation options

Ternary Operator

The ternary operator provides a shorthand way to write simple if-else statements.

let age = 20;
let status = age >= 18 ? "adult" : "minor";
console.log(`You are an ${status}`);

You are an adult

Switch Statement

Use switch statements when you have many conditions to check against a single value.

let day = "Monday";

switch (day) {
  case "Monday":
    console.log("Start of the work week");
    break;
  case "Friday":
    console.log("TGIF!");
    break;
  case "Saturday":
  case "Sunday":
    console.log("Weekend!");
    break;
  default:
    console.log("Regular work day");
}

Start of the work week

1.4 Loops

Learn how to repeat code using loops - a fundamental concept in programming that allows you to execute code multiple times.

What You'll Learn

  • For loops and while loops
  • Loop control with break and continue
  • Iterating over arrays and objects
  • Nested loops
  • Common loop patterns

For Loop

The for loop is used to repeat code a specific number of times.

// Basic for loop
for (let i = 0; i < 5; i++) {
  console.log(`Count: ${i}`);
}

Count: 0
Count: 1
Count: 2
Count: 3
Count: 4

For...of Loop

The for...of loop is used to iterate over iterable objects like arrays.

// Loop through an array
let fruits = ["apple", "banana", "orange"];
for (let fruit of fruits) {
  console.log(fruit);
}

apple
banana
orange

For...in Loop

The for...in loop is used to iterate over object properties.

// Loop through object properties
let person = {
  name: "Alice",
  age: 25,
  city: "New York"
};

for (let key in person) {
  console.log(`${key}: ${person[key]}`);
}

name: Alice
age: 25
city: New York

While Loop

The while loop repeats code as long as a condition is true.

let count = 1;
while (count <= 5) {
  console.log(`Count: ${count}`);
  count++;
}

Count: 1
Count: 2
Count: 3
Count: 4
Count: 5

Do...While Loop

The do...while loop executes the code block at least once, then checks the condition.

let number = 1;
do {
  console.log(`Number: ${number}`);
  number++;
} while (number <= 3);

Number: 1
Number: 2
Number: 3

Break and Continue

Use break to exit a loop early and continue to skip the current iteration.

// Using break
for (let i = 1; i <= 10; i++) {
  if (i === 6) {
    break;
  }
  console.log(i);
}

1
2
3

// Using continue
for (let i = 1; i <= 5; i++) {
  if (i === 3) {
    continue;
  }
  console.log(i);
}

1
2
4
5

Nested Loops

You can nest loops inside other loops to create complex patterns.

// Creating a multiplication table
for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    console.log(`${i} x ${j} = ${i * j}`);
  }
}

1 x 1 = 1
1 x 2 = 2
1 x 3 = 3
2 x 1 = 2
2 x 2 = 4
2 x 3 = 6
3 x 1 = 3
3 x 2 = 6
3 x 3 = 9

  • Use for loops when you know the number of iterations
  • Use while loops when you don't know the number of iterations
  • Use for...of to iterate over arrays
  • Use for...in to iterate over object properties
  • Use break to exit loops early

JavaScript Functionality

Now let's implement the JavaScript logic to make our calculator functional:

// script.js
document.addEventListener('DOMContentLoaded', function() {
  // Get DOM elements
  const billInput = document.getElementById('bill');
  const tipSelect = document.getElementById('tip');
  const peopleInput = document.getElementById('people');
  const tipAmountSpan = document.getElementById('tip-amount');
  const totalAmountSpan = document.getElementById('total-amount');
  const perPersonSpan = document.getElementById('per-person');

  // Add event listeners
  billInput.addEventListener('input', calculateTip);
  tipSelect.addEventListener('change', calculateTip);
  peopleInput.addEventListener('input', calculateTip);

  // Calculate tip function
  function calculateTip() {
    // Get input values
    const bill = parseFloat(billInput.value) || 0;
    const tipPercent = parseFloat(tipSelect.value) || 0;
    const people = parseInt(peopleInput.value) || 1;

    // Calculate tip and total
    const tipAmount = (bill * tipPercent) / 100;
    const totalAmount = bill + tipAmount;
    const perPerson = totalAmount / people;

    // Update display
    tipAmountSpan.textContent = `$${tipAmount.toFixed(2)}`;
    totalAmountSpan.textContent = `$${totalAmount.toFixed(2)}`;
    perPersonSpan.textContent = `$${perPerson.toFixed(2)}`;
  }

  // Initialize calculation
  calculateTip();
});

3.2 Event Listeners

Event listeners allow JavaScript to respond to user interactions and browser events. This is essential for creating interactive web applications.

What You'll Learn

  • Mouse events (click, hover, drag)
  • Keyboard events (keydown, keyup)
  • Form events (submit, change, input)
  • Browser events (load, resize, scroll)
  • Event object and properties

Basic Event Listeners

Use addEventListener() to attach event handlers to elements.

const button = document.getElementById('myButton');
button.addEventListener('click', function() {
  console.log('Button clicked!');
});

Mouse Events

Common mouse events include click, dblclick, mouseenter, mouseleave, mousedown, mouseup.

const element = document.querySelector('.my-element');

element.addEventListener('click', handleClick);
element.addEventListener('mouseenter', handleMouseEnter);
element.addEventListener('mouseleave', handleMouseLeave);

function handleClick(event) {
  console.log('Element clicked!');
  event.target.style.backgroundColor = 'red';
}

Keyboard Events

Handle keyboard input with keydown, keyup, and keypress events.

document.addEventListener('keydown', function(event) {
  console.log('Key pressed:', event.key);
  console.log('Key code:', event.keyCode);

  if (event.key === 'Enter') {
    console.log('Enter key pressed!');
  }
});

Form Events

Handle form interactions with submit, change, and input events.

const form = document.getElementById('myForm');
const input = document.getElementById('myInput');

form.addEventListener('submit', function(event) {
  event.preventDefault();
  console.log('Form submitted!');
});

input.addEventListener('input', function(event) {
  console.log('Input value:', event.target.value);
});

Event Object

The event object contains information about the event that occurred.

element.addEventListener('click', function(event) {
  console.log('Event type:', event.type);
  console.log('Target element:', event.target);
  console.log('Mouse position:', event.clientX, event.clientY);
  console.log('Prevented default:', event.defaultPrevented);
});

Preventing Default Behavior

Use preventDefault() to stop the browser's default action.

const link = document.querySelector('a');
link.addEventListener('click', function(event) {
  event.preventDefault();
  console.log('Link clicked but navigation prevented');
});

Event Bubbling and Capturing

Events bubble up through the DOM tree. Use stopPropagation() to prevent this.

const parent = document.getElementById('parent');
const child = document.getElementById('child');

parent.addEventListener('click', function() {
  console.log('Parent clicked');
});

child.addEventListener('click', function(event) {
  console.log('Child clicked');
  event.stopPropagation(); // Prevents parent from being notified
});

3.3 Fetch API

The Fetch API provides a modern way to make HTTP requests and handle responses in JavaScript.

What You'll Learn

  • Making GET and POST requests
  • Handling JSON responses
  • Error handling and status codes
  • Working with external APIs
  • Async/await with fetch

Basic GET Request

Use fetch() to make HTTP requests. It returns a Promise.

fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log('Data received:', data);
  })
  .catch(error => {
    console.error('Error:', error);
  });

Async/Await with Fetch

Use async/await for cleaner, more readable code.

async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log('Data:', data);
  } catch (error) {
    console.error('Error:', error);
  }
}

POST Request

Send data to a server using POST requests.

async function postData() {
  const data = {
    name: 'John Doe',
    email: 'john@example.com'
  };

  const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(data)
  });

  const result = await response.json();
  console.log('Response:', result);
}

async function fetchWithErrorHandling() {

    }

    const data = await response.json();
    return data;
  } catch (error) {

3.4 Error Handling

Proper error handling is crucial for building robust JavaScript applications that don't crash when things go wrong.

What You'll Learn

  • Promise error handling
  • Async/await error handling

Try-Catch Blocks

Use try-catch to handle errors gracefully.

try {
  const result = someFunction();
  console.log('Success:', result);

}

Error Types

JavaScript has different types of errors that you can catch and handle.

try {
  const data = JSON.parse('invalid json');
} catch (error) {
  if (error instanceof SyntaxError) {
    console.log('Invalid JSON format');
  } else {
    console.log('Other error:', error.message);
  }
}

Custom Errors

Create custom error classes for specific error types.

class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = 'ValidationError';
  }
}

function validateAge(age) {
  if (age < 0 || age > 120) {
    throw new ValidationError('Age must be between 0 and 120');
  }
  return age;
}

Promise Error Handling

Handle errors in promises using .catch() or try-catch with async/await.

fetch('https://api.example.com/data')
  .then(response => {
    if (!response.ok) {
      throw new Error('Network response was not ok');
    }
    return response.json();
  })
  .then(data => console.log('Data:', data))
  .catch(error => console.error('Error:', error));

Async/Await Error Handling

Use try-catch with async functions for cleaner error handling.

async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Failed to fetch data:', error.message);
    return null;
  }
}
