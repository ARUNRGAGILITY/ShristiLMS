Comments

Comments are used to explain code and make it more readable. Python supports single-line and multi-line comments.

graph TD A[Python Comments] --> B[Single Line #] A --> C[Multi-line '''] A --> D[Docstrings] B --> E[Inline Comments] B --> F[Block Comments] C --> G[Triple Single Quotes] C --> H[Triple Double Quotes] D --> I[Function Documentation] D --> J[Class Documentation]
Python Comments Examples 
# This is a single-line comment
print("Hello, World!")  # This is an inline comment

# Multi-line comment using triple quotes
"""
This is a multi-line comment
that spans multiple lines
"""

'''
Another way to write
multi-line comments
'''

def greet(name):
    """
    This is a docstring - used to document functions
    
    Args:
        name (str): The name to greet
    
    Returns:
        str: A greeting message
    """
    return f"Hello, {name}!"
Output
Hello, World!
üí° Best Practice: Use comments to explain WHY you're doing something, not WHAT you're doing. Good code should be self-explanatory about what it does.

Input & Output

Learn how to get input from users and display output in Python.

flowchart LR A[User Input] --> B[input() function] B --> C[Process Data] C --> D[print() function] D --> E[Display Output] F[File Input] --> G[File Processing] G --> H[File Output]
Input and Output Examples 
# Basic input and output
name = input("Enter your name: ")
print("Hello,", name)

# Input with type conversion
age = int(input("Enter your age: "))
print(f"You are {age} years old")

# Multiple inputs
first_name = input("First name: ")
last_name = input("Last name: ")
print(f"Full name: {first_name} {last_name}")

# Formatted output
price = 19.99
quantity = 3
total = price * quantity
print(f"Total: ${total:.2f}")

# Multiple print parameters
print("Name:", name, "Age:", age, sep=" | ")
print("Python", "is", "awesome!", sep="-")
Output
Enter your name: Alice Hello, Alice Enter your age: 25 You are 25 years old First name: John Last name: Doe Full name: John Doe Total: $59.97 Name: Alice | Age: 25 Python-is-awesome!

Variables

Variables are containers for storing data values. Python has dynamic typing, meaning you don't need to declare variable types explicitly.

graph TB A[Python Variables] --> B[Dynamic Typing] A --> C[Naming Rules] A --> D[Assignment] B --> E[Type Inference] B --> F[Runtime Type Checking] C --> G[Must start with letter or _] C --> H[Can contain letters, numbers, _] C --> I[Case sensitive] D --> J[Single Assignment] D --> K[Multiple Assignment] D --> L[Chained Assignment]
Variable Examples 
# Variable assignment
name = "Alice"
age = 25
height = 5.6
is_student = True

# Multiple assignment
x, y, z = 1, 2, 3
print(f"x={x}, y={y}, z={z}")

# Same value to multiple variables
a = b = c = 10
print(f"a={a}, b={b}, c={c}")

# Variable naming examples
first_name = "John"  # Good: snake_case
_private_var = "secret"  # Good: private variable
counter2 = 0  # Good: contains numbers

# Variable types can change
var = 42        # int
var = "Hello"   # str
var = 3.14      # float
var = True      # bool

# Checking variable type
print(type(name))
print(type(age))
print(type(height))
print(type(is_student))
Output
x=1, y=2, z=3 a=10, b=10, c=10 <class 'str'> <class 'int'> <class 'float'> <class 'bool'>
‚ö†Ô∏è Warning: Variable names cannot start with numbers, contain spaces, or use Python keywords like 'if', 'for', 'while', etc.

Data Types

Python has several built-in data types. Understanding these is crucial for effective programming.

graph TD A[Python Data Types] --> B[Numeric Types] A --> C[Text Type] A --> D[Boolean Type] A --> E[Sequence Types] A --> F[Mapping Type] A --> G[Set Types] A --> H[None Type] B --> B1[int] B --> B2[float] B --> B3[complex] C --> C1[str] D --> D1[bool] E --> E1[list] E --> E2[tuple] E --> E3[range] F --> F1[dict] G --> G1[set] G --> G2[frozenset] H --> H1[NoneType]
Data Types Examples 
# Numeric types
integer_num = 42
float_num = 3.14159
complex_num = 3 + 4j

# Text type
text = "Hello, Python!"
multi_line = """This is a
multi-line string"""

# Boolean type
is_true = True
is_false = False

# Sequence types
my_list = [1, 2, 3, 4, 5]
my_tuple = (1, 2, 3, 4, 5)
my_range = range(1, 6)

# Mapping type
my_dict = {"name": "Alice", "age": 25}

# Set types
my_set = {1, 2, 3, 4, 5}
my_frozenset = frozenset([1, 2, 3, 4, 5])

# None type
nothing = None

# Type checking and conversion
print(f"integer_num: {type(integer_num)} = {integer_num}")
print(f"float_num: {type(float_num)} = {float_num}")
print(f"complex_num: {type(complex_num)} = {complex_num}")
print(f"text: {type(text)} = {text}")
print(f"is_true: {type(is_true)} = {is_true}")

# Type conversion
str_num = "123"
converted_int = int(str_num)
converted_float = float(str_num)
print(f"Original: {str_num} ({type(str_num)})")
print(f"To int: {converted_int} ({type(converted_int)})")
print(f"To float: {converted_float} ({type(converted_float)})")
Output
integer_num: <class 'int'> = 42 float_num: <class 'float'> = 3.14159 complex_num: <class 'complex'> = (3+4j) text: <class 'str'> = Hello, Python! is_true: <class 'bool'> = True Original: 123 (<class 'str'>) To int: 123 (<class 'int'>) To float: 123.0 (<class 'float'>)

Best Practices

  • Follow PEP 8 for code style
  • Use meaningful variable names
  • Keep functions small and focused
  • Use type hints for better code documentation
  • Handle exceptions appropriately
  • Use docstrings to document functions and classes
  • Prefer composition over inheritance
  • Use virtual environments for project isolation
  • Write tests for your code
  • Use logging instead of print statements for debugging
